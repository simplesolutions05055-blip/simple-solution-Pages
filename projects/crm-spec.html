מבוא ומטרות

מערכת ה-CRM של Simple Solution היא פלטפורמת ניהול קשרי לקוחות מבוססת ענן, המאפשרת לבעלי עסק ולצוותי מכירות לנהל לקוחות, לידים, פעילויות ותהליכי מכירה ממקום אחד, באמצעות ממשק עברי, נקי, מהיר ונגיש מכל מכשיר.

M

חזון

להעניק לעסקים קטנים-בינוניים שליטה מלאה בתיק הלקוחות שלהם, ללא תלות במערכות יקרות ומסורבלות.

G

יעדים עסקיים

  • קיצור זמן טיפול בליד ב-40%.
  • שיפור שיעור המרה (conversion) ב-15%.
  • תיעוד מלא ושקוף של כל אינטראקציה.
K

מדדי הצלחה (KPI)

  • DAU/MAU גבוה מ-40%.
  • שביעות רצון משתמשים (NPS) ≥ 45.
  • זמן טעינת עמוד מתחת ל-2.5 שניות.
היקף
המסמך מגדיר את MVP (שלב א') ואת המודולים המתקדמים בשלב ב' ו-ג'. ראו פרק Roadmap להפרדה.

קהלי יעד ו-Personas

מערכת ה-CRM פונה לארבעה פרסונות עיקריות, ולכל אחת צרכים, יכולות ורמות גישה שונות.

1. בעל עסק (Owner)

מנהל את כלל הפעילות, רוצה תמונה כללית ושליטה בהגדרות וההרשאות.

  • צופה בלוח בקרה כללי וב-KPIs.
  • מנהל משתמשים והרשאות.
  • מקבל התראות על עסקאות גדולות.

2. מנהל מכירות (Sales Manager)

מנהל צוות מכירות, עוקב אחר ביצועים ומחלק לידים.

  • מקצה לידים לנציגים.
  • בודק דוחות ביצועים שבועיים.
  • מאשר הצעות מחיר חריגות.

3. נציג מכירות (Sales Rep)

איש השטח/הטלפון — המשתמש העיקרי במערכת ביום-יום.

  • עובד בעיקר במובייל.
  • מתעד שיחות ומשימות.
  • מקדם לידים בפייפליין.

4. עובד תפעול / שירות (Read-Only / Support)

צופה במידע לצורך מתן שירות, ללא הרשאות עריכה גורפות.

  • צפייה בכרטיס לקוח.
  • הוספת הערות.
  • פתיחת קריאות שירות.

פונקציות מרכזיות במודול הלקוחות

  • רשימת לקוחות עם חיפוש חופשי, סינון לפי תגית/סטטוס/מנהל תיק וסידור עמודות מותאם.
  • יצירה / עריכה / השבתה (Soft Delete) של לקוח, כולל בדיקת כפילויות לפי ח.פ/אימייל.
  • כרטיס לקוח חד-עמוד עם Tabs: סקירה, אנשי קשר, פעילויות, הצעות מחיר, מסמכים, הערות, היסטוריה.
  • שיוך מנהל תיק (Owner) ושינוי בעלות עם רישום ב-Audit Log.
  • ייצוא לקובץ CSV / Excel של רשומה בודדת או של רשימה מסוננת.

4.2 אנשי קשר (Contacts)

איש קשר מקושר ללקוח (Many-to-One). כולל שם, תפקיד, טלפון, אימייל, הערות והגדרת איש קשר ראשי.

  • כל לקוח חייב לפחות איש קשר ראשי אחד.
  • תמיכה במספר טלפונים/אימיילים לאיש קשר אחד.
  • סנכרון אופציונלי עם אנשי קשר ב-Google / Outlook (שלב ב').

4.4 משימות ותזכורות (Tasks)

  • יצירת משימה משויכת ללקוח/ליד/איש קשר, עם תאריך יעד, עדיפות (נמוכה/רגילה/דחופה).
  • תזכורות לפני מועד היעד (15 דק' / שעה / יום).
  • תצוגת "המשימות שלי" — קלנדר, רשימה ולוח Kanban (לעשות / בעבודה / הושלם).
  • השלמת משימה רושמת אוטומטית פעילות ב-Timeline של הלקוח.

4.5 תיעוד פעילויות (Activity Timeline)

כל אינטראקציה עם לקוח נשמרת בציר זמן אחיד: שיחות, מיילים, פגישות, הערות, שינויי סטטוס, שליחת הצעות.

  • הוספה ידנית עם סוג פעילות, משך, תוצאה וטקסט חופשי.
  • תיעוד אוטומטי של אירועי מערכת (יצירה, שינוי שלב, שליחת מסמך).
  • סינון לפי סוג / משתמש / טווח תאריכים.

4.6 מסמכים והערות

  • העלאת קבצים (PDF, DOC, XLSX, תמונות) עד 25MB לקובץ; אחסון ב-Object Storage עם חתימת URL.
  • הערות עשירות (Rich-Text קל) עם תיוג משתמשים @דנה.
  • סיווג מסמכים: חוזה / הצעת מחיר / חשבונית / אחר.

4.7 הצעות מחיר וחשבוניות (שלב ב')

  • בונה הצעת מחיר עם פריטים, כמויות, הנחות, מע"מ, תנאי תשלום.
  • ייצוא ל-PDF ממותג + שליחה במייל ישירות מהמערכת.
  • סטטוסים: טיוטה / נשלחה / אושרה / נדחתה / פג תוקף.
  • חיבור עתידי לחשבוניות ירוקות / Hashavshevet / iCount.

4.8 לוח בקרה ודוחות (Dashboard)

  • Widgets ניתנים להזזה: לידים פתוחים, ערך פייפליין, יחס סגירה, משימות מתקרבות.
  • גרפים: הכנסה חודשית, פילוח מקורות לידים, ביצועים לפי נציג.
  • שמירת תצוגות אישיות וייצוא PDF/Excel.

4.9 חיפוש גלובלי וסינונים מתקדמים

  • חיפוש אחד (⌘K) על פני לקוחות, לידים, אנשי קשר ומסמכים, עם דירוג רלוונטיות.
  • בונה סינונים: AND/OR, שמירת סינון מועדף, שיתוף עם צוות.

4.10 התראות

  • התראות in-app בפעמון + סיכום יומי במייל.
  • טריגרים: שיוך משימה, אזכור (mention), הזדמנות גדולה נכנסה, הצעת מחיר פגה.
  • הגדרת ערוצים והשתקה (Mute) לפי שעות / סוג.

4.11 הגדרות מערכת

  • שדות מותאמים אישית (Custom Fields) לישויות עיקריות.
  • ניהול תגיות, צוותים, שלבי פייפליין, סיבות אובדן.
  • תבניות מייל וחתימות.
  • מיתוג: לוגו, צבע ראשי, פרטי חברה.

User Stories וזרימות מפתח

דוגמאות מייצגות לזרימות המרכזיות. כל סיפור מנוסח בפורמט As a … I want … so that ….

US-01 · יצירת לקוח חדש

As a נציג מכירות I want ליצור לקוח חדש מהר ככל הניתן so that אוכל להתחיל לטפל בו עוד באותה השיחה.

  • Form עם שדות חובה מינימליים (שם, טלפון או אימייל).
  • בדיקת כפילויות בזמן הקלדה.
  • שמירה ומעבר ישיר לכרטיס הלקוח.

US-02 · קידום ליד בפייפליין

As a נציג I want לגרור ליד בין שלבים so that אוכל לעדכן סטטוס בלי לפתוח טפסים.

  • Drag-and-drop עם אישור ויזואלי.
  • אם המעבר ל"נסגר אבד" — דורש סיבת אובדן.
  • רישום אוטומטי ב-Activity Timeline.

US-03 · התראת תזכורת

As a נציג I want לקבל תזכורת 15 דקות לפני פגישה so that לא אשכח לקוח.

  • Push בדפדפן + מייל.
  • אפשרות לדחות (Snooze) ב-10/30 דק'.

US-04 · דוח חודשי למנהל

As a מנהל מכירות I want לייצא דוח חודשי של ביצועי הצוות so that אוכל להציגו בישיבת הנהלה.

  • בחירת טווח תאריכים, נציגים, מקורות.
  • ייצוא PDF + Excel.
  • שמירת תבנית דוח לשימוש חוזר.

עקרונות UX / UI

RTL-first ועברית קודם

כל הרכיבים מתוכננים מימין לשמאל מהיסוד, כולל אייקונים, חצים, גרפים וטבלאות. אנגלית תיתמך כשלב ב'.

Mobile responsive

נציג מכירות עובד בעיקר במובייל — כל מסך מרכזי חייב להיות שמיש בגודל 375px ומעלה.

Design Tokens

שימוש בטוקנים מרכזיים (צבעים, מרווחים, רדיוסים, צללים) לעקביות בין מסכים ולתחזוקה קלה.

נגישות (WCAG 2.1 AA)

ניגודיות מינימלית, ניווט מקלדת מלא, תוויות ARIA, פוקוסים נראים, תמיכה בקוראי מסך בעברית.

מהירות תפיסה

Skeletons בטעינה, Optimistic UI בפעולות עריכה, אנימציות עדינות בלבד (≤ 200ms).

Dark mode

תמיכה במצב כהה החל משלב ב', עם החלפה אוטומטית לפי הגדרת מערכת ההפעלה.

ארכיטקטורה טכנית מומלצת

המלצה מנומקת; הצוות הטכני רשאי להחליף רכיב — כל עוד עומדים בדרישות הלא-פונקציונליות.

חשבונית ירוקה / iCount

הפקת חשבונית מס מהצעת מחיר אושרה (שלב ב'–ג').

אבטחה ופרטיות

  • אימות: Supabase Auth, סיסמאות חזקות, אפשרות ל-MFA (TOTP).
  • אכיפת הרשאות במסד: Postgres Row Level Security — נציג רואה רק את הלקוחות שהוא Owner שלהם.
  • הצפנה: TLS 1.3 בתעבורה, AES-256 במנוחה (ברירת מחדל של Supabase).
  • Audit Log מלא לכל שינוי משמעותי: יצירה, עריכה, מחיקה, ייצוא, שינוי הרשאה.
  • פרטיות: תאימות לחוק הגנת הפרטיות הישראלי ול-GDPR — זכות עיון, תיקון ומחיקה (Right to be forgotten).
  • גיבויים: יומיים אוטומטיים ב-Supabase, שמירה 30 יום; שחזור Point-in-Time (שלב ב').
  • סודות: ניהול ב-Vercel Environment Variables + Supabase Vault, ללא commit ל-Git.
  • Rate-limiting על endpoints ציבוריים, ניטור התקפות brute-force.

Lead
